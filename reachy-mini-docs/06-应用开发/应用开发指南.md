# 应用开发指南

## 概述

Reachy Mini 有一个由 Hugging Face Spaces 驱动的应用生态系统。您可以构建 Python 应用，发布它们，任何 Reachy Mini 所有者都可以从仪表板一键安装。

## 应用工作原理

Reachy Mini daemon 管理应用的整个生命周期:

1. 您发送 "启动应用" 请求 (从仪表板或 REST API)
2. Daemon 将您的应用作为 **Python 子进程** 启动
3. 您的应用接收已连接的 `ReachyMini` 实例和 `stop_event`
4. 停止时，daemon 发送 `SIGINT` 触发优雅关闭
5. 应用退出后，daemon 将机器人恢复到默认位置

**关键约束:**
- 同时只能运行 **一个应用**
- 您的应用在 daemon 的子进程中运行 - 它不管理自己的硬件连接
- 在 Wireless 机器人上，您的应用运行在共享虚拟环境 `/venvs/apps_venv/` 中

## 创建应用

始终使用 CLI 工具创建应用 - 它生成正确的结构、元数据和入口点:

```bash
# 安装 reachy-mini (如果尚未安装)
uv pip install reachy-mini

# 创建并发布 (推荐)
reachy-mini-app-assistant create my_app_name /path/to/destination --publish

# 或先本地创建
reachy-mini-app-assistant create my_app_name /path/to/destination
```

> **永远不要手动创建应用文件夹。** 助手处理样板代码、Hugging Face 标签、入口点和正确的包结构。手动创建会导致难以调试的微妙问题。

### 选择模板

| 模板 | 命令 | 使用场景 |
|------|------|----------|
| **默认** | `reachy-mini-app-assistant create my_app .` | 大多数应用。最小工作结构。 |
| **对话** | `reachy-mini-app-assistant create --template conversation my_app .` | LLM 集成、语音、让机器人说话。包含音频管道、LLM 工具、运动融合和所有管道。 |

### 生成的结构

```
my_app/
├── index.html              # Hugging Face Space 登录页
├── style.css               # 登录页样式
├── pyproject.toml          # 包配置和入口点
├── README.md               # 必须包含 reachy_mini_python_app 标签
└── my_app/
    ├── __init__.py
    ├── main.py             # 您的应用逻辑
    └── static/             # 可选 Web UI
        ├── index.html
        ├── style.css
        └── main.js
```

## ReachyMiniApp 协议

您的应用是扩展 `ReachyMiniApp` 的类，实现 `run()` 方法:

```python
import threading
import time
import numpy as np

from reachy_mini import ReachyMini, ReachyMiniApp
from reachy_mini.utils import create_head_pose

class MyApp(ReachyMiniApp):
    def run(self, reachy_mini: ReachyMini, stop_event: threading.Event):
        t0 = time.time()

        while not stop_event.is_set():
            t = time.time() - t0

            # 移动头部
            yaw = 30.0 * np.sin(2.0 * np.pi * 0.2 * t)
            head_pose = create_head_pose(yaw=yaw, degrees=True)

            # 移动天线
            a = np.deg2rad(25.0 * np.sin(2.0 * np.pi * 0.5 * t))
            antennas = np.array([a, -a])

            reachy_mini.set_target(head=head_pose, antennas=antennas)
            time.sleep(0.02)

if __name__ == "__main__":
    app = MyApp()
    try:
        app.wrapped_run()
    except KeyboardInterrupt:
        app.stop()
```

### 关键点

- **`run(reachy_mini, stop_event)`** - 唯一必须实现的方法。`ReachyMini` 实例已连接并准备就绪。在主循环中轮询 `stop_event` 以优雅退出。
- **`wrapped_run()`** - 从 `__main__` 块调用。它处理连接到机器人、启动可选服务和调用您的 `run()` 方法。
- **`stop()`** - 设置 `stop_event`。Daemon 通过 `SIGINT` 调用此方法停止应用。
- **`__main__` 块** - 必需。Daemon 将您的应用作为模块运行 (`python -m my_app.main`)，因此此块是实际入口点。

## pyproject.toml 入口点

Daemon 通过标准 Python 入口点发现您的应用:

```toml
[project.entry-points."reachy_mini_apps"]
my_app = "my_app.main:MyApp"
```

组名是 `reachy_mini_apps` (带下划线)。值指向您的类 (`module.main:ClassName`)。

项目的任何额外依赖都应添加在此文件中。

## 可选: 应用的 Web UI

如果您想为应用设置设置页面或任何 Web 界面，在类上设置 `custom_app_url`:

```python
class MyApp(ReachyMiniApp):
    custom_app_url: str | None = "http://0.0.0.0:8042"

    def run(self, reachy_mini: ReachyMini, stop_event: threading.Event):
        # 在 self.settings_app 上定义 FastAPI 路由
        @self.settings_app.post("/my_endpoint")
        def my_endpoint():
            return {"status": "ok"}

        # 您的主循环...
```

设置 `custom_app_url` 后，应用会自动启动 FastWeb 服务器，从包内的 `static/` 目录提供文件。仪表板显示设置图标以打开此 UI。

设置 `custom_app_url = None` 如果您的应用不需要 Web UI。

## 测试应用

### 1. 验证结构

```bash
reachy-mini-app-assistant check /path/to/my_app
```

### 2. 直接运行

```bash
python -m my_app.main
```

### 3. 通过仪表板测试

```bash
# 以开发模式安装
uv pip install -e /path/to/my_app

# 启动 daemon
reachy-mini-daemon          # Lite
reachy-mini-daemon --sim    # 仿真
```

然后打开 http://127.0.0.1:8000/ - 您的应用出现在已安装列表中。

## 发布到 Hugging Face

### 1. 登录 Hugging Face

```bash
uv pip install --upgrade huggingface_hub
hf auth login
```

使用具有 **Write** 权限的令牌。

### 2. 发布

如果创建时使用了 `--publish`，它已经是 Hugging Face Space，带有 Git 远程。只需推送:

```bash
git add . && git commit -m "my changes" && git push
```

如果创建时没有 `--publish`，可以稍后发布:

```bash
reachy-mini-app-assistant publish /path/to/my_app
```

### 3. 可发现性

要让您的应用出现在 Reachy Mini 应用商店中，其 `README.md` 必须在 YAML 前言中包含 `reachy_mini_python_app` 标签:

```yaml
---
tags:
  - reachy_mini_python_app
---
```

助手会自动添加此标签。

## 安装应用

### 从仪表板

打开 Reachy Mini 仪表板，点击任何社区应用的 **Install**。

### 通过 REST API

```bash
# 从 Hugging Face 安装
curl -X POST http://localhost:8000/api/apps/install \
  -H "Content-Type: application/json" \
  -d '{"url": "https://huggingface.co/spaces/<user>/<app_name>"}'

# 启动应用
curl -X POST http://localhost:8000/api/apps/start-app/<app_name>

# 停止当前应用
curl -X POST http://localhost:8000/api/apps/stop-current-app

# 列出已安装应用
curl http://localhost:8000/api/apps/list
```

### 离线/手动部署 (Wireless)

```bash
# 复制并安装应用到机器人
scp -r /path/to/my_app pollen@reachy-mini.local:/tmp/my_app
ssh pollen@reachy-mini.local "/venvs/apps_venv/bin/pip install /tmp/my_app"
```

## 调试应用

### 查看日志

```bash
# 在终端运行 daemon，应用日志会直接显示
reachy-mini-daemon          # Lite
reachy-mini-daemon --sim    # 仿真

# Wireless: 通过 SSH 查看日志
ssh pollen@reachy-mini.local
sudo journalctl -u reachy-mini-daemon -f

# 最近日志，过滤
sudo journalctl -u reachy-mini-daemon --since '5 min ago' | grep -v "uvicorn\|GET \|POST "
```

### 常见问题

| 问题 | 解决方案 |
|------|----------|
| "An app is already running" | 先停止当前应用 |
| Daemon 状态异常 | 重启 daemon |
| 应用不拾取代码更改 | 重启应用 |

## 应用配置

应用子进程 **继承 daemon 的环境变量**。没有特殊的注入机制。

推荐的配置方法:
- **Web UI:** 设置 `custom_app_url`，让用户从浏览器输入值
- **配置文件:** 从已知路径读取 (例如 `.env`)
- **硬编码默认值:** 简单且可调试
