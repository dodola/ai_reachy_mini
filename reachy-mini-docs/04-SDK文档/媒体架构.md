# 媒体架构

## 统一架构

Reachy Mini 的 daemon 始终通过 `GstMediaServer` 管理摄像头和音频硬件，无论是 Reachy Mini (Wireless) 还是 Reachy Mini Lite。这意味着两种型号的工作方式相同:

- **Daemon** 拥有物理摄像头和音频设备
- **本地客户端** (同一台机器) 从本地 IPC 端点读取摄像头帧，直接通过 GStreamer 打开音频设备 - `LOCAL` 后端
- **远程客户端** 通过 WebRTC 从 daemon 流式传输摄像头 + 音频 - `WEBRTC` 后端
- SDK 根据 daemon 的 IPC 端点是否可达自动检测使用哪个后端

## Daemon 端

Daemon 除非传递 `--no-media` 标志，否则会自动启动媒体管道。它:

1. 打开摄像头 (平台感知: v4l2, libcamera, DirectShow, AVFoundation, 或仿真用 UDP)
2. 打开音频设备 (平台感知: PulseAudio, ALSA, WASAPI, CoreAudio)
3. 将两者输入 WebRTC 服务器 (`webrtcsink`) 用于远程流
4. 通过本地 IPC 端点暴露原始摄像头帧 (Linux/macOS 用 `unixfdsink`, Windows 用 `win32ipcvideosink`)

## 客户端端

SDK 的 `MediaManager` 自动选择后端:

| 后端 | 说明 | 使用场景 |
|------|------|----------|
| **LOCAL** | 客户端与 daemon 在同一台机器时使用。从 IPC 读取摄像头帧，直接通过 GStreamer 打开音频设备。无编解码开销。 | 本地开发 |
| **WEBRTC** | 客户端远程时使用。通过 WebRTC 流式传输摄像头 + 音频。 | 远程连接 |
| **NO_MEDIA** | 跳过所有媒体初始化。 | 无头操作 |

## Web 访问

得益于 WebRTC，音频和视频流也可以直接从 Web 浏览器访问。例如桌面应用就使用了这个功能。

## 禁用媒体 / 直接硬件访问

默认情况下，daemon 拥有摄像头和音频设备。如果您需要直接访问硬件 - 例如使用 OpenCV、sounddevice 或自定义视觉管道 - 您可以停用内置的媒体管理器:

```python
from reachy_mini import ReachyMini

with ReachyMini(media_backend="no_media") as mini:
    # daemon 已释放摄像头和音频硬件。
    # 直接使用 OpenCV、sounddevice 或任何其他库。
    import cv2
    cap = cv2.VideoCapture(0)
    ret, frame = cap.read()
    cap.release()

    # 机器人控制仍然正常工作。
    mini.goto_target(antennas=[0.3, -0.3], duration=0.5)

# 退出时，daemon 自动重新获取硬件。
```

当传递 `media_backend="no_media"` 时，SDK:
1. 请求 daemon **释放**摄像头和音频设备 (停止 GStreamer 管道)
2. 将本地 `MediaManager` 设置为 `NO_MEDIA` (不通过 SDK 获取摄像头/音频)
3. 在上下文管理器退出时 (`__exit__`)，告诉 daemon **重新获取**硬件

### 手动控制

您也可以随时手动调用 `release_media()` 和 `acquire_media()`:

```python
mini = ReachyMini()

# ... 使用内置媒体管理器 ...
frame = mini.media.get_frame()

# 切换到直接访问
mini.release_media()
# ... 使用 OpenCV、sounddevice 等 ...

# 切换回 SDK 媒体管理器
mini.acquire_media()
frame = mini.media.get_frame()
```

> **注意:** 两个方法都是幂等的 - 调用 `release_media()` 两次是安全的。

## 音频使用

### 录制

```python
with ReachyMini(media_backend="default") as mini:
    # 初始化
    mini.media.start_recording()
    
    # 录制
    samples = mini.media.get_audio_sample()
    # 返回形状 (samples, 2) 的 float32 数组，采样率 16kHz
    
    # 停止
    mini.media.stop_recording()
```

### 播放

```python
with ReachyMini(media_backend="default") as mini:
    # 初始化
    mini.media.start_playing()
    
    # 播放
    mini.media.push_audio_sample(audio_data)
    # 期望形状 (samples, 1 或 2) 的 float32 数组，采样率 16kHz
    
    # 停止
    mini.media.stop_playing()
```

### 声源定位 (DoA)

```python
with ReachyMini(media_backend="default") as mini:
    mini.media.start_recording()
    
    # 获取声源方向
    # 0 弧度 = 左, π/2 弧度 = 前/后, π 弧度 = 右
    doa, is_speech_detected = mini.media.get_DoA()
    
    mini.media.stop_recording()
```

## 媒体后端选项

| 后端 | 说明 | 使用场景 |
|------|------|----------|
| `default` | 自动检测最佳后端 | 大多数情况 |
| `local` | 强制 LOCAL 后端 | 同一台机器 |
| `webrtc` | 强制 WEBRTC 后端 | 远程连接 |
| `no_media` | 禁用媒体管理器 | 直接硬件访问 |

> **提示:** 对于大多数设置，后端会根据您是在本地还是远程运行自动选择。无需指定 `media_backend` 值！

> **提示:** WebRTC 后端需要在客户端机器上安装 GStreamer。
