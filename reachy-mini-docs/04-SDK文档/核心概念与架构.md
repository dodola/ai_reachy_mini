# 核心概念与架构

## 软件架构

Reachy Mini 使用 **客户端-服务器** 架构:

### 1. Daemon (服务器)
- 运行在连接机器人的计算机上 (或仿真环境)
- 处理硬件 I/O (USB/串口)、安全检查和传感器读取
- 暴露 REST API (`localhost:8000`) 和 WebSocket

### 2. SDK (客户端)
- 您的 Python 代码 (`reachy_mini` 包)
- 通过网络连接到 Daemon
- **优势:** 您可以在强大的服务器上运行 AI 代码，而 Daemon 运行在连接机器人的 Raspberry Pi 上

## 坐标系

移动机器人时，您将使用两个主要参考系:

### 1. 头部坐标系 (Head Frame)

位于头部底座。用于 `goto_target` 和 `set_target` 命令。

```
        Z (向上)
        |
        |
        |_________ Y (向前)
       /
      /
     /
    X (向右)
```

**轴说明:**
- **X:** 左右平移
- **Y:** 前后平移
- **Z:** 上下平移
- **Roll:** 绕 Y 轴旋转
- **Pitch:** 绕 X 轴旋转
- **Yaw:** 绕 Z 轴旋转

### 2. 世界坐标系 (World Frame)

相对于机器人底座固定。用于 `look_at_world` 命令。

```
        Z (向上)
        |
        |
        |_________ Y (向前)
       /
      /
     /
    X (向右)
```

## 安全限制

Reachy Mini 有物理和软件限制，以防止自碰撞和损坏。SDK 会自动将值限制在最近的有效位置。

| 关节/轴 | 限制范围 |
|---------|----------|
| Head Pitch/Roll | [-40°, +40°] |
| Head Yaw | [-180°, +180°] |
| Body Yaw | [-160°, +160°] |
| Yaw Delta (头-身) | 最大 65° 差异 |

> **注意:** 超出限制的值会被自动限制，不会报错。

## 电机模式

您可以改变电机的行为方式:

```python
with ReachyMini() as mini:
    # 僵硬模式 - 保持位置
    mini.enable_motors()
    
    # 松弛模式 - 无动力，可手动移动
    mini.disable_motors()
    
    # 重力补偿模式 - "柔软"，可手动移动并保持位置
    mini.enable_gravity_compensation()
```

### 模式说明

| 模式 | 说明 | 使用场景 |
|------|------|----------|
| `enable_motors()` | 电机开启，保持位置 | 需要机器人保持姿态 |
| `disable_motors()` | 电机关闭，无动力 | 手动移动机器人 |
| `enable_gravity_compensation()` | 电机开启但"柔软" | 示教编程 |

## 运动控制

### goto_target() vs set_target()

| 方法 | 说明 | 使用场景 |
|------|------|----------|
| `goto_target()` | 平滑插值，有加速/减速 | 手势、动画 (≥0.5秒) |
| `set_target()` | 即时设置目标 | 实时控制、追踪 (≥10Hz) |

### 插值方法

| 方法 | 说明 |
|------|------|
| `linear` | 线性插值 |
| `minjerk` | 最小加加速度 (默认，最自然) |
| `ease_in_out` | 缓入缓出 |
| `cartoon` | 卡通风格 (快速移动，突然停止) |

## 媒体架构

### 统一架构

Daemon 始终通过 `GstMediaServer` 管理摄像头和音频硬件:

- **Daemon** 拥有物理摄像头和音频设备
- **本地客户端** (同一台机器) 从本地 IPC 端点读取摄像头帧 - `LOCAL` 后端
- **远程客户端** 通过 WebRTC 流式传输摄像头 + 音频 - `WEBRTC` 后端
- SDK 自动检测使用哪个后端

### 后端选择

| 后端 | 说明 | 使用场景 |
|------|------|----------|
| `LOCAL` | 本地 IPC，无编解码开销 | 同一台机器 |
| `WEBRTC` | WebRTC 流 | 远程连接 |
| `NO_MEDIA` | 禁用媒体 | 无头操作 |

### 直接硬件访问

如果需要直接访问硬件 (例如使用 OpenCV):

```python
with ReachyMini(media_backend="no_media") as mini:
    import cv2
    cap = cv2.VideoCapture(0)
    ret, frame = cap.read()
    cap.release()
    
    # 机器人控制仍然正常工作
    mini.goto_target(antennas=[0.3, -0.3], duration=0.5)

# 退出时，daemon 自动重新获取硬件
```

## 连接模式

### 自动检测 (推荐)

```python
with ReachyMini() as mini:
    # SDK 自动检测 Lite vs Wireless
    pass
```

### 手动指定

```python
# 本地连接 (Lite)
with ReachyMini(connection_mode="localhost_only") as mini:
    pass

# 网络连接 (Wireless)
with ReachyMini(connection_mode="network") as mini:
    pass
```

## 应用生命周期

1. Daemon 接收 "启动应用" 请求
2. Daemon 作为 Python 子进程启动应用
3. 应用接收已连接的 `ReachyMini` 实例和 `stop_event`
4. 停止时，Daemon 发送 `SIGINT` 触发优雅关闭
5. 应用退出后，Daemon 将机器人恢复到默认位置

**关键约束:**
- 同时只能运行一个应用
- 应用在 Daemon 的子进程中运行

## REST API

Daemon 暴露 HTTP/WebSocket API:

| 端点 | 说明 |
|------|------|
| `http://localhost:8000/docs` | API 文档 |
| `GET /api/state/full` | 获取完整状态 |
| `ws://localhost:8000/api/state/ws/full` | WebSocket 实时更新 |

### 使用场景
- Web UI
- 非 Python 客户端
- 远程控制
- AI/LLM 集成

## 平台差异

| 平台 | 计算 | 摄像头 | 说明 |
|------|------|--------|------|
| **Lite** | 完整 (笔记本) | 直接 USB | 最灵活，适合开发 |
| **Wireless (本地)** | 有限 (CM4) | 直接 | 内存/CPU 受限 |
| **Wireless (流式)** | 完整 (笔记本) | 通过网络 | 有一定延迟 |
| **仿真** | 完整 | 无 | 无法测试摄像头功能 |
