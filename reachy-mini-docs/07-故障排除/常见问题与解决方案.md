# 常见问题与解决方案

## 基本故障排除

**在做任何其他事情之前，请阅读本节！**

### 更新并重启

**确保您使用的是最新软件，并且已重启机器人和计算机。**

要重启机器人，按 OFF，等待 5 秒，然后按 ON。这个简单的过程修复了几个常见和已知问题。

### 如何更新软件

**使用 Reachy Mini Control:**
- 从 "⚙️" 设置标签，"Check for updates"
- 如果有更新可用，点击 "Update now"

**使用克隆的仓库:**
- 确保在最新的标签版本上，或
- 与 `main` 分支保持最新 (`git pull`)

**Wireless Reachy Mini:** 运行 `reachyminios_check` 确保一切正常

## 🛠️ 故障排除 - 常见问题

### Bootstrap 或更新失败 / Python 环境问题 (Lite & 仿真)

如果 Reachy Mini Control 在初始 bootstrap 期间失败，卡在创建虚拟环境，或更新使 Python 环境处于损坏状态，您可以从桌面应用重置虚拟环境。

**两种重置选项:**
- **重置应用环境** - 仅重新创建 `apps_venv`。已安装的应用需要重新安装。
- **完全环境重置** - 删除所有 Python 文件并重新下载所有内容。这可能需要几分钟。

**按钮位置:**
- **连接前** (Finding Robot 屏幕): 点击右上角的 ⚙️ 图标
- **连接后** (USB / 仿真模式): 打开 Settings overlay，转到 "Environment" 部分

### 电机相关问题

**症状:**
- 电机闪烁红色或不响应/移动
- 电机显示 "Overload Error" 等错误
- 缺少电机: "No motor found on port", "Missing motor" 等

**解决方案:**
参见 [电机诊断和故障排除指南](https://huggingface.co/docs/reachy_mini/troubleshooting/motors_diagnosis)

使用 Reachy Mini Testbench 应用识别和解决这些问题。

### 电击错误

Dynamixel 电机上的电击错误意味着电源有问题或 somewhere 有短路。

**检查:**
- 检查是否有电缆损坏，从脚 PCB 到头部
- 特别是以下电缆:
  - 电源线 (黑色和红色)
  - 电机的 3 线电缆 (300mm, 200mm, 100mm 和 40mm)

### 麦克风不工作

**症状:**
- 音频录制不工作 / 返回静音
- 音频录制返回零

**可能原因:**
- 麦克风电缆插反了

**检查:**
- 如果电缆是白色和蓝色的，请再次检查组装说明以确保蓝色面朝上
- 如果电缆是黑色的，检查 "Main Board" 的一侧朝上

**如果电缆正确但仍有问题:**
麦克风的 FPC 电缆可能损坏。参见 [如何更换 Reachy Mini 麦克风的 FPC 电缆?](https://huggingface.co/docs/reachy_mini/troubleshooting/change_mic_fpc_cable)

### 音频音量低

- 更新机器人到 1.2.3 或更高版本

### 权限错误

- 更新机器人到 1.2.3 或更高版本
- 重启机器人

### 天线旋转 90° 或 180°

这是制造问题。

按照此指南轻松修复:
[天线重新定位指南](https://drive.google.com/file/d/1FsmNpwELuXUbdhGHDMjG_CNpYXOMtR7A/view?usp=drive_link)

### 电机抖动

**天线抖动**

这是最常见的情况。天线 (电机 17 和 18) 在垂直位置 (0°) 时容易抖动。

**解决方案:**
1. 将天线偏移几度 (通常 10° 足够)
2. 调整 PID 控制值

**调整 PID:**
- 首先尝试将 P 降低到 180 (电机 10, 17, 18)
- 如果没有帮助，尝试将 D 增加到 10

### 头部运动时发出吱吱声或摩擦声

如果听到高频吱吱声或摩擦声，Stewart 平台连杆上的球形接头可能需要维护。

按照 [球形接头维护指南](https://huggingface.co/docs/reachy_mini/troubleshooting/spherical_joints_maintenance) 清洁和重新润滑接头。

### Lite 版本图像较暗

**快速修复: 调整摄像头设置中的曝光时间**

**推荐应用:**
- **macOS:** [CameraController](https://github.com/itaybre/CameraController)
- **Linux:** `sudo apt install qv4l2`
- **Windows:** [Webcam Settings](https://www.softpedia.com/get/Internet/WebCam/Webcam-Settings-Tool.shtml)

**高级:** 使用 libuvc 命令行工具设置 `auto-exposure-priority=1`

### 包中缺少零件

确保先拆开所有东西。一些零件是预组装的 (例如底部头部零件已经放在后头部零件中)。

如果确定缺少零件，请联系 sales@pollen-robotics.com，附上所有零件的照片和订单号或发票号。

### 无法使用 USB-C 电缆连接 Wireless Reachy Mini

Wireless 单元不像 Lite 版本那样通过 USB 暴露机器人。

**替代方案:**
- 将机器人加入 WiFi 网络，使用 SDK 客户端远程控制
- SSH 进入嵌入式 Raspberry Pi 并在那里执行脚本
- 使用 USB-C 转以太网适配器加以太网电缆直接连接

### WiFi 热点不显示 - RPI 不启动

检查头中板上的开关是否在 "debug" 位置 (不是 "download")。

如果开关位置正确但仍看不到 AP，可能需要重新刷写 Raspberry Pi ISO。

## 🚀 入门和组装

### 组装需要多长时间?

大多数测试人员报告 **1.5 到 2 小时**。根据您的经验水平，最多可能需要 4 小时。

### 组装过程中有困难步骤吗?

没有，测试人员描述为 **有趣、简单和令人满意**。基本工具和耐心就足够了。**电缆布线** 和 **正确拧紧零件** 是最棘手的元素。

### 组装后还剩 2 根电缆和几个螺丝。这正常吗?

是的，这完全正常。我们故意在套件中包含备用电缆和螺丝，以防组装过程中损坏或丢失零件。

### Reachy Mini 首次启动时不动。应该检查什么?

* **电源:** 确保 7V-5A 电源已插入。USB 连接不足以驱动电机。
* **电缆:** 检查所有电缆是否完全插入。松动的电源线是 "电机不响应" 错误的常见原因。

### 需要手动启动 daemon 吗?

**不需要**
- Reachy Mini (Wireless): daemon 已在嵌入式 Raspberry Pi 上运行
- Reachy Mini Lite: 可以使用桌面应用

## 🔌 连接和 Reachy Mini Control

### 如何将机器人连接到 WiFi?

参见 [Reachy Mini Wireless 指南](https://huggingface.co/docs/reachy_mini/platforms/reachy_mini/get_started)

### 如何重置 WiFi 热点?

按照 [WiFi 重置指南](https://huggingface.co/docs/reachy_mini/platforms/reachy_mini/reset) 中的说明操作。

### 机器人有 Web API 吗?

是的。daemon 提供 REST API (FastAPI) 和 WebSocket 支持。
* **文档:** `http://localhost:8000/docs` (当 daemon 运行时)
* **功能:** 获取状态、移动关节、控制 daemon

**WebSocket 支持:**
```javascript
let ws = new WebSocket(`ws://127.0.0.1:8000/api/state/ws/full`);
ws.onmessage = (event) => {
    const data = JSON.parse(event.data);
    console.log(data);
};
```

### 为什么需要虚拟环境 (.venv)?

帮助防止 SDK 安装期间的包冲突。

### reachy-mini.local 不解析

Wireless 单元通过 mDNS 将自己广播为 `reachy-mini.local`。这在大多数家庭和办公网络上有效，但在某些企业、会议或酒店网络上可能失败。

**解决方案:**
- 检查路由器的 DHCP 客户端列表
- 使用 Reachy Mini Control 应用
- 扫描子网

### 机器人和计算机在会议/酒店 WiFi 上无法通信

许多会议和酒店 WiFi 启用 **客户端隔离**。

**解决方法:** 使用手机热点。

### 从中国访问 HuggingFace 服务?

使用镜像: https://hf-mirror.com/

```bash
export HF_ENDPOINT=https://hf-mirror.com/
```

### 如何让对话应用在中国工作?

Reachy Mini 对话应用依赖 OpenAI gpt-realtime API。

**最佳解决方法:**
1. 在您的机器 (Lite 版本)、机器人 (Wireless 版本) 或路由器上设置 VPN
2. 配置 VPN 路由规则:
   - 通过 VPN 路由流量，除了本地网络流量
   - 白名单: 本地 LAN IP 范围、端口 22, 8000, 5353

## 🤖 硬件、电机和限制

### 在哪里可以找到更多硬件信息?

参见 [Reachy Mini 硬件页面](https://huggingface.co/docs/reachy_mini/platforms/reachy_mini/hardware)

### 安全限制是什么 (头部和身体)?

* **身体 Yaw:** [-180°, 180°]
* **头部 Pitch/Roll:** [-40°, 40°]
* **头部 Yaw:** [-180°, 180°]
* **组合限制:** `body_yaw` 和 `head_yaw` 之间的差异必须在 **[-65°, 65°]** 以内

### 为什么电机 "松弛" 或 "僵硬"? (合规性)

* **`enable_motors()`**: 电机 **开启** (僵硬)。机器人保持位置。
* **`disable_motors()`**: 电机 **关闭** (松弛)。您可以手动移动它。
* **`make_motors_compliant()`**: 电机 **开启但柔软**。用于示教编程。

### 如何访问电机参数?

1. 使用 [scan_motors.py 脚本](https://github.com/pollen-robotics/reachy_mini/tree/main/src/reachy_mini/tools/scan_motors.py) 扫描电机

**Lite 版本:**
```bash
python -m reachy_mini.tools.scan_motors
```

**Wireless 版本:**
```bash
ssh pollen@reachy-mini.local
source /venvs/mini_daemon/bin/activate
python -m reachy_mini.tools.scan_motors --wireless
```

正常输出:
```
Found motors at baudrate 1000000: [10, 11, 12, 13, 14, 15, 16, 17, 18]
```

### 电机过一段时间后停止响应

* 检查电源连接
* 电机可能进入热保护模式 (过热)。关闭后再打开
* 更新 SDK (`pip install -U reachy-mini`)

### 电池有安全功能吗?

Wireless 包含 proper 电池充电器。电池还集成了带温度传感器的 BMS。

### 如何查看剩余电量?

无法检查电池状态，这是设计的已知限制。只有 "低电量" 的 LED 指示。

### 如何取出电池

1. 检查绿色 LED 是否不亮
2. 取下底部的 3 颗螺丝，稍微取出脚
3. 拔下指示的连接器以取出电池
4. 重新组装时，按相反顺序进行

### 头部在某些官方动作期间可能接触身体

这是预期行为，不是硬件或软件错误。

### 我可以修改外观 (皮肤/CAD) 吗?

* **CAD:** 完整的 STEP 文件尚未发布，但计划在未来发布。目前可以在仓库的 [assets 文件夹](https://github.com/pollen-robotics/reachy_mini/tree/develop/src/reachy_mini/descriptions/reachy_mini/mjcf/assets) 中找到一些 STL 文件。
* **皮肤:** 是的，社区创建了自定义构建。一些 "皮肤" STEP 文件已在 Discord 的 ["Pimp my Reachy Mini"](https://discord.com/channels/519098054377340948/1453340883775651861) 频道分享。

### 我的麦克风扁平电缆坏了

规格:
- FFC/FPC 扁平柔性电缆
- 12 pins
- 0.5mm 间距
- Type A (连接器在同一侧)
- 15mm 长度

**替换参考:**
- [Amazon](https://www.amazon.fr/dp/B09TR4X1BP)
- [Farnell](https://fr.farnell.com/molex/15020-0127/cordon-ffc-12-cond-152mm-blanc/dp/3862090)

## 🕹️ 移动机器人

### 如何移动头部?

```python
from reachy_mini.utils import create_head_pose

with ReachyMini() as mini:
    mini.goto_target(head=create_head_pose(yaw=-10, pitch=20))
```

### goto_target 和 set_target 有什么区别?

* **`goto_target`**: **平滑**。随时间插值运动 (默认 0.5 秒)。最适合手势。
* **`set_target`**: **即时**。立即设置目标。最适合高频控制 (遥操作、数学轨迹)。

### 如何录制和回放动作?

**录制:**
```python
mini.start_recording()
# ... 移动机器人 ...
move = mini.stop_recording()
```

**回放:**
```python
from reachy_mini.motion.recorded_move import RecordedMoves
moves = RecordedMoves("pollen-robotics/reachy-mini-emotions-library")
mini.play_move(moves.get("happy"))
```

### 机器人的动作看起来抖动。控制循环运行正确吗?

检查 daemon 状态:
```python
print(mini.client.get_status())
```

您应该看到大约 50Hz (~20ms 周期) 的值。

如果周期远大于 20ms，可能是因为:
- 计算机上的 CPU 负载过高
- (仅限 Lite) USB 延迟高

## 👁️ 视觉和音频

### Linux 音量太低

```bash
alsamixer
# 设置 PCM1 到 100%
# 使用 PCM,0 调整全局音量

# 永久保存
CARD=$(aplay -l | grep -i "reSpeaker" | head -n1 | sed -n 's/^card \([0-9]*\):.*/\1/p')
amixer -c "$CARD" set PCM,1 100%
sudo alsactl store "$CARD"
```

### 如何获取摄像头帧?

```python
with ReachyMini() as mini:
    frame = mini.media.get_frame()
    # 返回 numpy 数组 (BGR 格式，兼容 OpenCV)
```

### 如何使用麦克风/扬声器?

```python
# 获取音频
sample = mini.media.get_audio_sample()

# 播放音频
mini.media.push_audio_sample(numpy_chunk)
```

### 如何让 Reachy 看某个东西?

* **2D (图像):** `mini.look_at_image(x, y)` - (0,0 是左上角)
* **3D (世界):** `mini.look_at_world(x, y, z)` - 机器人坐标系中的坐标

### 面部追踪感觉慢

性能 heavily 依赖于光照条件。确保面部光线充足。

### 如何检查声音系统是否工作?

**Lite 版本:**
直接从您的计算机使用 *Pollen Robotics Reachy Mini Audio* 设备验证功能。

**Wireless 版本:**
```bash
# 录制声音
gst-launch-1.0 -e alsasrc device="reachymini_audio_src" ! audioconvert ! audioresample ! wavenc ! filesink location="test.wav"

# 播放录制
gst-launch-1.0 filesrc location=test.wav ! wavparse ! audioconvert ! alsasink device=reachymini_audio_sink

# 播放测试声音 (粉红噪声)
gst-launch-1.0 audiotestsrc wave="pink-noise" ! audioconvert ! audioresample ! alsasink device="reachymini_audio_sink"
```

## 🔧 特定错误消息和修复

### 电机 '' hardware errors: ['Input Voltage Error']

我们在 Reachy Mini 上使用更高的电压，这是故意的 :)

### Error: "OSError: PortAudio library not found"

安装系统依赖:
```bash
sudo apt-get install libportaudio2
```

### Warning: "Circular buffer overrun" (仿真/Mujoco)

如果连接到机器人但不消费视频帧，缓冲区会填满。

**修复:** 如果不需要视频，使用 `ReachyMini(media_backend="no_media")` 初始化。

### 对话应用在更新后崩溃 (自定义配置文件)

**症状:** 日志以 `SystemExit: 1` 结束

**修复 (推荐):** 从 Reachy Mini Control 桌面应用点击 **Reset apps environment**

**替代 (高级):**
```bash
rm -rf /venvs/apps_venv
# 然后重新安装对话应用
```

### 没有麦克风输入 / 声源方向 (Beta 单元)

* **无输入:** 需要固件 2.1.3。运行 [更新脚本](https://github.com/pollen-robotics/reachy_mini/tree/main/src/reachy_mini/assets/firmware/update.sh)
* **无方向:** 需要固件 2.1.0+
